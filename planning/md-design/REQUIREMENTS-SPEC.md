# Requirements Spec

## 1. Actors
| Actor | Description |
|---|---|
| Primary User | 신한카드 Excel 명세서를 업로드하고 소비 패턴을 분석하는 사용자 |
| Optional Admin | 이번 MVP에서는 제외하고 단일 사용자로 처리 |

---

## 2. Main Use Cases

### UC-001. 명세서 업로드 및 파싱
- **Actor:** Primary User
- **Goal:** 신한카드 Excel 명세서를 업로드해 거래 내역을 자동으로 불러온다
- **Precondition:** 사용자가 신한카드 Excel 명세서 파일(.xlsx)을 보유하고 있다
- **Main Flow:**
  1. 사용자가 명세서 업로드 화면에서 Excel 파일을 선택한다
  2. 시스템이 파일을 파싱해 거래 내역(날짜, 가맹점명, 금액)을 추출한다
  3. Claude API가 가맹점명을 분석해 카테고리를 자동 분류한다
  4. 분류된 데이터가 대시보드에 표시된다
- **Alternative Flow:**
  - 파일 형식이 잘못된 경우 에러 메시지를 표시하고 재업로드를 안내한다
  - 가맹점명이 모호한 경우 "기타"로 분류한다
- **Result:** 거래 내역이 카테고리별로 분류되어 대시보드에 표시된다

### UC-002. 대시보드 확인
- **Actor:** Primary User
- **Goal:** 월간 또는 연간 단위로 기간별 지출 현황을 카드 목록으로 파악한다
- **Precondition:** 명세서가 업로드되어 거래 내역이 존재한다
- **Main Flow:**
  1. 사용자가 월간 / 연간 토글을 선택한다
  2. 시스템이 명세서 내 해당 단위의 기간 카드 목록을 최신순으로 표시한다
  3. 각 카드에 해당 기간의 총 지출액(취소 제외)과 최다 지출 카테고리가 표시된다
  4. 사용자가 기간 카드를 클릭하면 기간 상세 화면으로 이동한다
- **Alternative Flow:**
  - Session Storage에 데이터가 없을 경우 빈 상태 안내 메시지를 표시한다
- **Result:** 기간별 지출 카드 목록이 표시되고, 특정 기간 상세로 진입할 수 있다

### UC-003. 기간 상세 확인
- **Actor:** Primary User
- **Goal:** 특정 기간의 전체 결제 내역과 카테고리별 지출을 상세히 확인한다
- **Precondition:** 대시보드에서 기간 카드를 클릭한 상태다
- **Main Flow:**
  1. 시스템이 해당 기간의 도넛 차트를 표시한다
  2. 기본 탭(전체 결제 내역)에 날짜 내림차순 거래 목록이 표시된다
  3. 취소 거래는 별도 색상과 배지로 구분 표시된다
  4. 사용자가 카테고리별 지출 탭을 선택하면 카테고리별 합계·비율 목록이 표시된다
  5. 사용자가 카테고리의 펼치기 버튼을 클릭하면 해당 카테고리 거래 내역이 인라인으로 펼쳐진다
- **Alternative Flow:**
  - 해당 기간에 데이터가 없을 경우 안내 메시지를 표시한다
- **Result:** 특정 기간의 전체 결제 내역과 카테고리별 상세 내역이 표시된다

### UC-004. 기간 보기 전환
- **Actor:** Primary User
- **Goal:** 월간 또는 연간 단위로 보기를 전환해 원하는 기간 단위로 소비를 확인한다
- **Precondition:** 명세서가 업로드되어 거래 내역이 존재한다
- **Main Flow:**
  1. 사용자가 대시보드 상단의 월간 / 연간 토글을 선택한다
  2. 시스템이 선택한 단위에 맞는 기간 카드 목록으로 즉시 갱신한다
- **Alternative Flow:**
  - 선택한 단위에 데이터가 없을 경우 빈 상태 안내 메시지를 표시한다
- **Result:** 선택한 기간 단위에 해당하는 카드 목록이 대시보드에 표시된다

## 3. Functional Requirements
| ID | Requirement | Priority |
|---|---|---|
| FR-001 | 사용자는 신한카드 Excel 명세서(.xlsx)를 업로드할 수 있다 | Must |
| FR-002 | 시스템은 업로드된 명세서에서 날짜, 가맹점명, 금액을 자동으로 추출한다 | Must |
| FR-003 | 시스템은 Claude API를 통해 가맹점명을 카테고리로 자동 분류한다 | Must |
| FR-004 | 사용자는 기간 카드 목록에서 각 기간의 총 지출액과 최다 지출 카테고리를 확인할 수 있다 | Must |
| FR-005 | 사용자는 기간 상세 화면에서 카테고리별 지출 비율을 도넛 차트로 확인할 수 있다 | Must |
| FR-006 | 사용자는 월간 / 연간 토글로 대시보드 기간 단위를 전환할 수 있다 | Must |
| FR-007 | 사용자는 기간 상세 화면에서 전체 결제 내역을 날짜 내림차순으로 확인할 수 있다 | Must |
| FR-008 | 사용자는 기간 상세 화면에서 카테고리별 지출 합계와 비율을 확인할 수 있다 | Must |
| FR-009 | 사용자는 카테고리별 지출 목록에서 펼치기 버튼으로 해당 카테고리의 상세 거래 내역을 인라인으로 확인할 수 있다 | Must |
| FR-013 | 취소 거래는 결제 내역 목록에 표시되되 금액 집계에서 제외되며 별도 색상으로 구분 표시된다 | Must |
| FR-010 | 파일 형식 오류 시 사용자에게 안내 메시지를 표시한다 | Should |
| FR-011 | 분류가 모호한 가맹점은 "기타"로 자동 분류한다 | Should |
| FR-012 | 사용자는 잘못 분류된 카테고리를 수동으로 수정할 수 있다 | Nice |

## 4. Non-functional Requirements
| ID | Requirement |
|---|---|
| NFR-001 | 모바일 화면에서도 핵심 기능을 사용할 수 있어야 한다 |
| NFR-002 | 버튼과 입력 필드는 접근 가능한 이름을 가져야 한다 |
| NFR-003 | 민감한 정보(API 키 등)는 GitHub 저장소에 커밋하지 않는다 |
| NFR-004 | 기본 MVP는 단일 사용자 기준으로 구현한다 |
| NFR-005 | 명세서의 개인정보는 서버에 저장하지 않고 세션 내에서만 처리한다 |
| NFR-006 | 명세서 업로드 후 대시보드 표시까지 5초 이내로 처리한다 |

## 5. Acceptance Criteria

### AC-001. 명세서 업로드
Given 사용자가 신한카드 Excel 파일을 선택했을 때  
When 업로드 버튼을 누르면  
Then 거래 내역이 파싱되어 카테고리가 분류된 대시보드가 표시된다

### AC-002. 대시보드 확인
Given 명세서가 업로드되어 데이터가 존재할 때  
When 사용자가 대시보드 화면을 열면  
Then 전체 지출 합계, 도넛 차트, 카테고리별 테이블이 표시된다

### AC-003. 드릴다운 상세 확인
Given 대시보드에 데이터가 표시된 상태일 때  
When 사용자가 특정 카테고리를 선택하면  
Then 해당 카테고리의 상세 지출 내역과 가맹점별 지출 정리가 표시된다

### AC-004. 기간 필터링
Given 명세서가 업로드되어 데이터가 존재할 때  
When 사용자가 기간 필터(월별 / 6개월 / 1년)를 선택하면  
Then 선택한 기간에 해당하는 데이터만 필터링되어 대시보드에 반영된다

### AC-005. 파일 오류 처리
Given 사용자가 잘못된 형식의 파일을 업로드했을 때  
When 업로드 버튼을 누르면  
Then 에러 메시지가 표시되고 재업로드를 안내한다
