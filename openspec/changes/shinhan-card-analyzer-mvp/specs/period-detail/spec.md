## ADDED Requirements

### Requirement: 사용자는 기간 카드를 클릭해 상세 내역을 확인할 수 있다
기간 카드 클릭 시 `/dashboard/[period]`로 이동해야 한다. `period` 파라미터 포맷은 월간 `YYYY-MM`, 연간 `YYYY`다. 상세 페이지에는 해당 기간의 도넛 차트와 탭 전환 UI가 표시되어야 한다.

#### Scenario: 월간 카드 클릭
- **WHEN** 사용자가 `2026-05` 월간 카드를 클릭하면
- **THEN** `/dashboard/2026-05`로 이동하고 5월 데이터가 표시된다

#### Scenario: 연간 카드 클릭
- **WHEN** 사용자가 `2026` 연간 카드를 클릭하면
- **THEN** `/dashboard/2026`으로 이동하고 2026년 데이터가 표시된다

### Requirement: 전체 결제 내역 탭은 시간순으로 거래를 표시한다
기본 탭은 `전체 결제 내역`이며, 해당 기간의 모든 거래를 날짜 내림차순으로 표시해야 한다. 취소 거래는 `--ctp-overlay-1` 컬러로 구분 표시하고 `취소` 배지를 함께 표시해야 한다.

#### Scenario: 전체 내역 기본 표시
- **WHEN** 기간 상세 페이지에 진입하면
- **THEN** `전체 결제 내역` 탭이 활성화되고 날짜 내림차순 거래 목록이 표시된다

#### Scenario: 취소 거래 구분 표시
- **WHEN** 취소 거래가 목록에 포함되면
- **THEN** 해당 행은 `--ctp-overlay-1` 컬러로 표시되고 `취소` 배지가 붙는다

### Requirement: 카테고리별 지출 탭은 인라인 accordion으로 상세를 펼친다
`카테고리별 지출` 탭 선택 시 카테고리별 합계와 비율을 목록으로 표시해야 한다. 각 카테고리 행의 펼치기 버튼 클릭 시 해당 카테고리의 거래 내역이 인라인으로 펼쳐져야 한다. 다른 카테고리를 동시에 펼칠 수 있어야 한다.

#### Scenario: 카테고리별 탭 전환
- **WHEN** 사용자가 `카테고리별 지출` 탭을 클릭하면
- **THEN** 카테고리별 지출 합계와 비율 목록이 표시된다

#### Scenario: 카테고리 accordion 펼치기
- **WHEN** 사용자가 특정 카테고리의 펼치기 버튼을 클릭하면
- **THEN** 해당 카테고리의 거래 내역이 인라인으로 펼쳐진다

#### Scenario: 여러 카테고리 동시 펼치기
- **WHEN** 사용자가 두 개의 카테고리를 순서대로 펼치면
- **THEN** 두 카테고리의 내역이 동시에 펼쳐진 상태로 표시된다

### Requirement: 도넛 차트는 카테고리별 지출 비율을 시각화한다
해당 기간의 카테고리별 지출 비율을 Recharts 도넛 차트로 표시해야 한다. 차트 색상은 Catppuccin 팔레트 순서(Mauve → Blue → Green → Peach → Teal → Yellow → Flamingo → Sky)를 따른다. 취소 거래는 차트 집계에서 제외해야 한다.

#### Scenario: 도넛 차트 렌더링
- **WHEN** 기간 상세 페이지에 진입하면
- **THEN** 카테고리별 비율 도넛 차트가 표시된다

#### Scenario: 취소 거래 차트 제외
- **WHEN** 해당 기간에 취소 거래가 있으면
- **THEN** 도넛 차트는 취소 거래를 제외한 금액 기준으로 비율을 계산한다
