## ADDED Requirements

### Requirement: 사용자는 신한카드 Excel 명세서를 업로드할 수 있다
시스템은 `.xlsx` 파일을 클릭 선택 또는 드래그앤드롭으로 업로드받아야 한다. `.xlsx` 이외의 파일 형식은 거부하고 안내 메시지를 표시해야 한다. 업로드 중에는 로딩 인디케이터를 표시하고 버튼을 비활성화해야 한다.

#### Scenario: 올바른 .xlsx 파일 업로드
- **WHEN** 사용자가 `.xlsx` 파일을 선택하고 업로드 버튼을 누르면
- **THEN** 파싱이 시작되고 로딩 인디케이터가 표시된다

#### Scenario: 잘못된 파일 형식 업로드
- **WHEN** 사용자가 `.csv` 또는 `.pdf` 등 `.xlsx` 이외의 파일을 선택하면
- **THEN** 에러 메시지가 표시되고 업로드가 진행되지 않는다

### Requirement: 시스템은 명세서에서 거래 내역을 추출한다
SheetJS를 사용해 클라이언트 사이드에서 파일을 파싱해야 한다. `거래일`, `가맹점명`, `금액` 컬럼을 추출해야 하며, 거래일은 `YYYY-MM-DD` 형식으로 정규화해야 한다. 금액 컬럼의 쉼표를 제거하고 정수로 변환해야 한다.

#### Scenario: 정상 파싱
- **WHEN** 유효한 신한카드 `.xlsx` 파일이 업로드되면
- **THEN** 거래 내역(날짜·가맹점·금액)이 추출되어 Transaction 배열로 반환된다

#### Scenario: 컬럼 구조 불일치
- **WHEN** 필수 컬럼(`거래일`, `가맹점명`, `금액`)이 없는 파일이 업로드되면
- **THEN** 에러 메시지를 표시하고 파싱을 중단한다

### Requirement: 시스템은 취소 거래를 감지하고 표시를 구분한다
다음 세 조건 중 하나라도 해당하면 취소 거래(`isCancel: true`)로 분류해야 한다: 금액이 음수, `매입구분`이 `승인취소`, `취소상태`가 `취소`. 취소 거래는 내역 목록에 포함하되 금액 집계에서 제외해야 하며, UI에서 `--ctp-overlay-1` 컬러로 구분 표시해야 한다.

#### Scenario: 취소 거래 감지 — 음수 금액
- **WHEN** 금액 컬럼 값이 음수인 행이 있으면
- **THEN** 해당 거래의 `isCancel`이 `true`로 설정된다

#### Scenario: 취소 거래 감지 — 매입구분
- **WHEN** `매입구분` 컬럼 값이 `승인취소`인 행이 있으면
- **THEN** 해당 거래의 `isCancel`이 `true`로 설정된다

#### Scenario: 취소 거래 집계 제외
- **WHEN** 취소 거래가 포함된 기간의 합계를 계산하면
- **THEN** `isCancel: true`인 거래의 금액은 합산되지 않는다

### Requirement: 시스템은 해외 거래를 포함해 처리한다
`거래통화`가 KRW가 아닌 해외 거래의 경우, `금액` 컬럼 값(이미 KRW 환산)을 그대로 사용해야 한다. 별도 환율 계산은 하지 않는다. `매입구분`이 `승인` 상태인 거래도 포함한다.

#### Scenario: 해외 거래 포함
- **WHEN** `거래통화`가 USD 또는 CNY인 거래가 명세서에 있으면
- **THEN** `금액` 컬럼의 KRW 환산값이 그대로 Transaction의 amount로 저장된다
