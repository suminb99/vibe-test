## ADDED Requirements

### Requirement: 사용자는 월간 또는 연간 보기로 전환할 수 있다
대시보드 상단에 `월간 | 연간` 토글이 있어야 한다. 선택된 모드에 따라 월별 카드 또는 연도별 카드 목록을 표시해야 한다. 기본값은 `월간`이다.

#### Scenario: 월간 모드 선택
- **WHEN** 사용자가 `월간` 토글을 선택하면
- **THEN** 명세서에 포함된 모든 월의 카드가 최신순으로 표시된다

#### Scenario: 연간 모드 선택
- **WHEN** 사용자가 `연간` 토글을 선택하면
- **THEN** 명세서에 포함된 연도별 카드가 최신순으로 표시된다

### Requirement: 각 기간 카드는 총 지출액과 최다 지출 카테고리를 표시한다
월간 카드는 해당 월의 취소 거래 제외 총 지출액과 가장 많이 지출한 카테고리명을 표시해야 한다. 연간 카드도 동일하게 연간 합계와 최다 카테고리를 표시해야 한다.

#### Scenario: 월간 카드 내용 표시
- **WHEN** 2026년 5월 카드가 렌더링되면
- **THEN** 5월 총 지출액(취소 제외)과 최다 지출 카테고리명이 카드에 표시된다

#### Scenario: 취소 거래 제외 합산
- **WHEN** 해당 월에 취소 거래가 포함되어 있으면
- **THEN** 카드에 표시되는 총액은 취소 거래를 제외한 금액이다

### Requirement: 데이터가 없을 때 빈 상태 화면을 표시한다
Session Storage에 `ParsedStatement`가 없을 때 대시보드 접근 시 업로드 안내 메시지와 업로드 페이지 이동 버튼을 표시해야 한다.

#### Scenario: 데이터 없음
- **WHEN** Session Storage에 데이터가 없는 상태로 `/dashboard`에 접근하면
- **THEN** 빈 상태 안내 메시지와 `/upload` 이동 버튼이 표시된다
